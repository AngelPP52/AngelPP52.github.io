            1. 线下编辑

          注意， 如果 MD 文档 抬头的格式错误， 可能导致编译失败， 导致无法看到您的文章。

          建议参照: 这篇文章 的抬头格式。

          # 1. 线上编辑

          打开目录：

          https://git.garena.com/shopee/bg-logistics/tech-article/-/tree/test/docs/views

          根据文章主题， 放在不同的目录下。

          文章用 markdown 语法编写， 易于扩展。

          如果不熟悉 markdown 语法， 可以参考 markdown 语法简介。

          # 2. 线下编辑

          仓库地址：

          https://git.garena.com/shopee/bg-logistics/tech-article/-/tree/test

          # 开启本地服务

          下载完成之后， 进入到文件目录， 执行：

          yarn && yarn dev

          如果没安装过 yarn， 可以参考 yarn 安装指南

          开启本地服务，点击生成的服务地址, (一般为: http://localhost:8081/), 即可预览博客首页。

          如果你是开发人员，可以本地启动 step 1 里介绍的服务，改动 markdown 文件内容，浏览器会立刻刷新， 实时预览。

          # 3. 关于图片

          文章的图片可以放到与你文章同级的img目录下。

          比如: 引入 img 目录下的的图片 demo.png:

          ![](./img/demo.png)
          1

          一篇文章内的图片可以放到单独的文件夹内。

          # 4. 推送代码

          文章完成之后, 把你的分支合并到 test 环境做进一步校验， 也可以邀请你的同事一起校对。

          校对完成之后，提交 mr 给对应的负责人， 负责人会合并到 master 分支, 并发布到 live。

          # 最后

          祝大家写作愉快。
